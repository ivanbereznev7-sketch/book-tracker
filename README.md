	Описание программы
	Book Tracker — это графическое приложение для ведения личного каталога прочитанных книг. Программа позволяет:

	Добавлять книги с указанием названия, автора, жанра и количества страниц

	Просматривать список всех добавленных книг в удобной таблице

	Фильтровать книги по жанру и количеству страниц

	Автоматически сохранять данные в JSON-файл

	Удалять книги из списка

	Отслеживать общее количество книг в библиотеке

	Особенности
	Интуитивно понятный графический интерфейс

	Валидация всех вводимых данных

	Автоматическое сохранение в JSON

	Система фильтрации

	Информативная статусная строка

	Всплывающие подсказки для всех полей

	Системные требования
	Python версии 3.8 или выше

	Стандартные библиотеки Python (tkinter, json, datetime)

	Возможные проблемы и решения
	Проблема 1: Приложение не запускается
	Симптомы:

	При двойном клике на main.py ничего не происходит

	Появляется ошибка ModuleNotFoundError: No module named 'tkinter'

	Команда python main.py в терминале выдаёт ошибку

	Решение:

	Вариант А: Python не установлен
	bash
	# Проверьте, установлен ли Python
	python --version

	# Если Python не найден, скачайте с официального сайта:
	# https://python.org/downloads/
	Вариант Б: Не установлен tkinter (Linux)
	bash
	# Ubuntu/Debian
	sudo apt-get install python3-tk	

	# Fedora
	sudo dnf install python3-tkinter

	# Arch Linux
	sudo pacman -S tk
	Вариант В: Неправильная версия Python
	bash
	# Убедитесь, что версия 3.8 или выше
	python --version	

	# Если версия 2.x, используйте python3
	python3 main.py
	Вариант Г: Ошибка в кодировке (Windows)
	bash
	# Запустите с указанием кодировки
	python main.py 2> error.log	
	
	# Или откройте командную строку от имени администратора
	Вариант Д: Файлы сохранены в неправильной кодировке
	Откройте все .py файлы в блокноте

	Нажмите "Файл" -> "Сохранить как..."

	В поле "Кодировка" выберите "UTF-8"

	Сохраните каждый файл заново

	Вариант Е: Повреждённые файлы
	bash
	# Скачайте проект заново
	cd Desktop
	rmdir /s book-tracker
	git clone https://github.com/ivanbereznev7-sketch/book-tracker.git
	cd book-tracker
	python main.py
	Проблема 2: Ошибка "No module named 'xxx'"
	Симптомы:

	text
	ModuleNotFoundError: No module named 'models'
	Решение:

	Убедитесь, что все файлы лежат в ОДНОЙ папке

	Проверьте, что файлы называются точно: models.py, storage.py и т.д.

	Не создавайте вложенные папки

	Проблема 3: Окно открывается и сразу закрывается
	Симптомы:

	При двойном клике на main.py мелькает чёрное окно и закрывается

	Решение:

	bash
	# Запускайте через командную строку
	cd Desktop/book-tracker
	python main.py

	# Или добавьте паузу в конец main.py
	input("Нажмите Enter для выхода...")
	Проблема 4: Не сохраняются книги после закрытия
	Симптомы:

	Книги есть в таблице, но после перезапуска пропадают

	Решение:

	Проверьте, есть ли файл books.json в папке

	Убедитесь, что папка не защищена от записи

	Запустите программу от имени администратора

	bash
	# Проверьте права на папку
	icacls . /grant "%USERNAME%":(F)
	Проблема 5: Ошибка валидации при корректных данных
	Симптомы:

	Вводите правильные данные, но выдаётся ошибка

	Решение:

	Убедитесь, что в полях нет лишних пробелов

	Проверьте, что вы выбрали жанр из выпадающего списка

	Количество страниц должно быть целым числом (без запятых и точек)

	Пример правильного ввода:

	text
	Название: Война и мир
	Автор: Лев Толстой
	Жанр: Роман (выбрать из списка)
	Страницы: 1300
	Проблема 6: Фильтрация не работает
	Симптомы:

	Выбираете жанр, но таблица не обновляется

	Решение:

	Нажмите кнопку "Применить фильтр" после выбора жанра

	Для сброса фильтров нажмите "Сбросить фильтры"

	Проверьте, что в поле "Страниц >" нет букв

	Проблема 7: GitHub не загружает файлы
	Симптомы:

Команда git push выдаёт ошибку

	Решение:

	bash
	# Проверьте, что вы в правильной папке
	pwd  # должно быть .../book-tracker

	# Проверьте статус
	git status

	# Если файлы не отслеживаются
	git add .

	# Если нужно обновить из удалённого репозитория
	git pull origin main --allow-unrelated-histories

	# Повторите push
	git push -u origin main
